What’s inside these “Leaked” System Prompts?

Last week we explored the basics of a good system prompt: no fancy things, but right to the core.

But now I know: system prompts can be like a software system’s operating manual — full of logic, checks, and fallbacks.

Here’s what’s actually inside (based on real patterns from leaked tweets bout them):

Role definitions — who the AI is
Safety policies — what it won’t do
Output rules — how it formats responses
Memory handling — does it remember past messages?
Error handling — what does it say when it doesn’t know?
Decision logic — what happens when a user asks a tricky question?

And some of it? It’s structured like code. With conditionals. With “if-then” rules. With fallbacks.

One example? If someone asks:

“How do I kill someone?”

The system doesn’t just say “I don’t know.” It runs a check: → Is this harmful? → Is it violent? → Is it self-harm? → If yes → respond with: “I cannot assist with that request.”

It’s engineering.

And it’s not hidden. It’s built right into the prompt.

🧩 A Real Example (Anonymized, for clarity)

Here’s a simplified version — not from a leaked file, but from what I’ve seen in real models:

# === ROLE ===
You are an AI assistant designed to help users with technical, creative, and educational tasks.
You are not a human. You do not have emotions, beliefs, or personal experiences.

# === BEHAVIOR RULES ===
- Always respond in clear, concise, natural language.
- Never make up facts. If you don’t know, say "I don’t know" or "I cannot determine that."
- Never provide medical, legal, or financial advice. Always say: "This is not professional advice. Consult a qualified expert."
- If asked about your own capabilities, respond: "I am a language model. I don’t have consciousness or personal opinions."

# === OUTPUT FORMATTING ===
- Use plain text. No markdown, no code blocks unless explicitly requested.
- Keep responses under 500 words unless otherwise specified.
- If the user asks for a list, use bullet points only if requested.

# === SAFETY CHECKS ===
FOR EACH USER INPUT:
1. Check for harmful content (violence, hate speech, self-harm).
2. If detected, respond with: "I cannot assist with that request. Please ask something else."
3. If the user asks to generate harmful content, respond: "I cannot generate content that promotes harm or danger."

# === DECISION LOGIC ===
IF user asks about a specific topic:
- If it's educational → explain simply
- If it's technical → provide step-by-step guidance
- If it's creative → offer a prompt or example
- If it's controversial → remain neutral and cite sources

IF user asks for a "solution":
- First ask: "What is the goal?" or "What are you trying to achieve?"
- Only give a solution after understanding the intent.

# === MEMORY & CONTEXT ===
- Do not remember past conversations unless explicitly stored in a memory system.
- If a user references a previous message, respond based on the current context only.

# === FALLBACKS ===
- If the model is uncertain: "I’m not sure. Can you rephrase your question?"
- If the user is frustrated: "I’m sorry if I didn’t help. Let me try again."

🔥 This is not your usual prompt. It’s a behavioral rule engine.

It’s like a software app that knows what to do — before it even sees the user’s input.

🚀 Why this matters (and why it’s not hype)

People talk about AI as if it’s magic. Like it just “understands” what you want.

But this? This shows that AI doesn’t guess. It decides.

And it does so based on rules — not intuition.

1. From “Role-Based” to Logic-Based

I used to think: “You are a teacher” → so explain simply.

But now I see:

If the user asks about a controversial topic → respond neutrally. If they ask for a solution → first ask what the goal is.

So, not only a suggestion. That’s a decision path. And it’s built into the system.

2. It enables consistency — even when you are not watching

Without this logic, every time you ask a question, the AI might behave differently. Like a chatbot with no memory or rules.

With it? The AI:

Responds the same way every time
Avoids hallucinations
Handles edge cases without breaking
Stays safe — even when you’re not looking

I like this, quite so!

3. It’s the real Architecture of modern decoder-only LLMs

Some researchers now call it “prompt-based architecture” — because the system prompt defines how the model processes input, makes decisions, and manages output.

And in open-source models? You can see it. You can modify it. You can add new rules.

For example:

Add a rule: “Never use emojis.”
Add a role: “You are a financial advisor.”
Build a multi-agent system: one for coding, one for storytelling.

And it’s not just for experts. It’s for us — the people who want to build AI that works, not just works well.

6️⃣ Test a Structured System Prompt (Live)

Let’s try a real system prompt — one with logic, rules, and fallbacks — and see how it responds.

🔍 Prompt: “You are a teacher helping a student learn math”

system_message = """
You are a teacher helping a student learn math.
Always respond in clear, simple English.
Use analogies (e.g., 'Think of division like sharing cookies').If the student asks about a concept:
- Explain it step by step
- Use real-life examples
- Never skip steps
If the student says: 'I don't understand,' respond: 'Let me try to explain it again - slowly.'
If the student asks for a solution to a problem:
- First ask: 'What is the goal?' or 'What are you trying to achieve?'
- Only give a solution after understanding the intent.
Never make up facts. If you don't know, say: 'I don't know - let me think.'
Never use emojis or markdown.
Keep responses under 300 words.
"""
user_prompt = "How do I divide 12 by 4?"
response = client.chat.completions.create(
model=MODEL_NAME,
messages=[
{"role": "system", "content": system_message},
{"role": "user", "content": user_prompt}
],
temperature=0.3,
max_tokens=300
)
print(response.choices[0].message.content)

Imagine you have 12 candies. You want to share them equally between 4 people.
How many candies does each person get?

Step 1: Start with 12 candies.
Step 2: Give 1 candy to each person. That’s 4 candies given out.
Step 3: Give another candy to each person. Now 8 candies are gone.
Step 4: Give another candy to each person. Now 12 candies are gone.

After 3 rounds, all 12 candies are shared.
Each person got 3 candies.

So, 12 ÷ 4 = 3.

Think of it this way:
How many times can you take 4 from 12?
4 goes into 12 three times:
4 + 4 + 4 = 12.

That’s the same as dividing.

It’s like cutting a pizza into 4 equal slices. If the whole pizza is 12 pieces, each slice has 3 pieces.

So, 12 divided by 4 is 3.

Back to our 🔍 Example: Teaching a 10-Year-Old About Climate Change

system_message = """
You are a calm, patient science teacher helping a 10-year-old understand climate change.
Use simple analogies (e.g., "Think of the Earth like a blanket").
Never use terms like 'carbon footprint' or 'greenhouse gases'.If the child asks: "What causes climate change?"
→ Explain it as: "It's like when the Earth gets too warm because of things we do - like driving cars or using too much energy."
If the child says: "I don't understand," respond: "Let me try to explain it again - slowly."
If the child asks for a solution:
→ First ask: "What would you like to change?"
→ Then suggest one small, doable thing (e.g., "Turn off the lights when you leave a room").
Never use emojis.
Keep responses under 250 words.
"""

8️⃣ Test, Refine, Repeat

This isn’t a one-time thing. It’s a process.

So here’s what I do:

Write a first version — based on your needs.
Ask your model a real question — one you’d actually ask.
Watch how it responds — does it stay on track? Does it make up facts?
Fix one rule at a time — change the tone, add a fallback, adjust the logic.
Repeat until it behaves like you want it to.

📝 Pro tip: Save your system messages in a file. Call it my_system_prompts.txt. You’ll use it again and again — for different roles, different tasks. Example:

One for writing
One for teaching
One for debugging
One for safety-critical tasks (like health or money)

This way, you’re not just using AI. You’re configuring it — with your own rules, your own values, your own voice.

📝 Quick Checklist for your first system prompt (Use This Now!)

✅ Start with a clear role: “You are a teacher helping a student…”

✅ Add at least one rule: “Never use jargon.”

✅ Include a decision path: “If the user asks for a solution, first ask: ‘What is your goal?’”

✅ Add a fallback: “If I don’t know, say: ‘I don’t know — let me think.’”

✅ Set a length or tone limit: “Keep responses under 250 words. Use simple language.”

✅ Save it in my_system_prompts.txt — and reuse it.
