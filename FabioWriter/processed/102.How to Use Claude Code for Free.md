# How to Use Claude Code for Free

## Want to use Claude Code without burning through local GPU resources (or your wallet)? Use it with Ollama Cloud integration.


Claude Code is a mega-popular vibe-coding environment, and Ollama is a great tool for making LLMs easy to use locally, for free.

Combining the power of Claude Code with the freedom of Ollama has got to result in something good.

To run Claude Code conventionally, you need to subscribe to the tune of $20 a month (less if you pay a year up front). Frankly, I don’t think that’s anything to complain about, given the productivity gains you could achieve.

But free is better!

Of course, Claude Code is not an LLM in itself. It’s an application, albeit a clever one, that builds on top of Anthropic LLMs. And as such, there ought to be a way of getting it to use another model. Anthropic has built in the hooks to enable you to do this, but you don’t need to delve into that because Ollama have already done it for you.

Ollama provides a simple integration that lets you use their models with Claude Code.

`ollama launch claude`

And if you have a suitably powerful computer, you can run a capable model such as Kimi or Deepseek Coder on your own hardware, and you are good to go.

If, like me, your hardware is not up to running these large models, there is still a solution. You can run Ollama models in the Ollama Cloud and, while you wouldn’t expect that to be free, there is a fairly good free quota that you can take advantage of.

The command below launches Claude Code with a specified model. Suffix that model with :`cloud` and you will be running the model in the Ollama Cloud. Here’s how you would run Kimi:

`ollama launch claude --model kimi-k2.5:cloud`

This setup gives you the best of both worlds—powerful remote AI models with a local, developer-friendly workflow. Let’s walk through how to get it running.

## Prerequisites and Setup

Before anything else, make sure your environment is ready:

-   [Install Claude Code CLI](https://code.claude.com/docs/en/overview) for your operating system
-   [Install Ollama](https://ollama.com/download)
-   Log in to Ollama Cloud by typing `ollama signin` in a terminal window

Run this in your terminal:

ollama login

That’s it—you’re now connected to Ollama’s cloud infrastructure.

## Running Claude Code

We are ready to go. Launch Claude Code with the following command in a terminal window:

`ollama launch claude --model kimi-k2.5:cloud`

Or use another of Ollama’s recommendations:

-   `kimi-k2.5:cloud`
-   `glm-5:cloud`
-   `minimax-m2.7:cloud`
-   `qwen3.5:cloud`

If you have the oomph available, pull the appropriate model and use that instead of the cloud-based models.

After invoking Claude Code, you will be asked if you trust the folder that you are using (presumably, yes) and then presented with something similar to the following screen:

![](https://miro.medium.com/v2/resize:fit:700/1*v-RglWKcBgjT2OpYl5uh4w.png)

You are now set to go to work. I’ve given it a simple three-sentence specification to construct an HTML/JavaScript version of Conway’s Game of Life. This is a 0-player game that simulates simple life forms.

![](https://miro.medium.com/v2/resize:fit:700/1*7-LkvdIuePG09QBvRq4ypA.png)

Having thought for a few minutes, Claude generates the code and asks if you want it to go ahead and write the file.

![](https://miro.medium.com/v2/resize:fit:700/1*c85F9VkYqdfgDlmDZtf7XA.png)

Which, of course, we do.

And we get a very nice, simple web app that does even more than we asked for: a version of the Game of Life with various user controls, in a very retro green-screen style (which you can see at the top of this article). _For interest, here is a_ [_link to the app_](https://alanjones2.github.io/conway/retro/game-of-life.html)_._

## It’s not just about coding

I gave Claude Code a very loose specification, and it came up with a suitable solution. Fine for a fun app like this, although for a serious program, we would have to provide a much tighter spec.

To refine an app, we could, of course, keep prompting until we have exactly what we want. Alternatively, we could write a tighter spec in the first place and take advantage of a CLAUDE.md file, where we can be very specific about how we define the work to be done.

To be competitive, the Ollama Cloud models would need to be effective over a range of related files in a project. And it is not just about coding: we ought to be able to produce documentation, write tests, commit files to a GitHub repo and so on.

The main point is that Claude Code seems to work well with Ollama Cloud models. How it would cope with a more complex task, I’ve yet to find out (but watch this space).
