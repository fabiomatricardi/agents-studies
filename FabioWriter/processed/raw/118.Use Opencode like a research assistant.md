# Use Opencode like a research assistant

Opencode is a great open source alternative to Claude code and other coding agent solutions. But if it is your only LLM application you use, it does not cover everything you might want from an AI solution like research and keeping track of it. But with a simple additional agent you can also have that right in your terminal.

This article is probably not interesting for people with an active ChatGPT, Claude or similar subscription. But for people who use LLMs 99% of the time as a coding tool within Opencode, this idea might come in handy.

## What is even the problem?

Open code is great. The built in tools and agents make it great as a coding agent. But if you also want to research some ideas there are some challenges with that. Both built in agents don’t feel great with open research questions, the build agent will most likely focus too much on building it and also the plan agent will mainly try to make a implementation plan for the problem. That’s totally fair, as these are their main purposes, but if you want to have a tool which should help you in finding the right solution to a hard problem, you want to have a different kind of conversation.

The second problem is, that you can’t really come back to a conversation. Sure, you can try to find the old session, but Opencode does not keep any memory of your old discussions by default (which is good for its main purpose)

## A simple way to solve this

Opencode already has all the tools a great research assistant needs built in. Websearch and other tools give it everything it needs from that perspective. To add the memory and the right way the tools talks with you can be done by the following setup:

-   Create a new folder where you use opencode from now on for research topics
-   Add an agent to the .opencode/agents folder within the folder, which has the instruction to function as a helpful research assistant, who also stores a summary for every topic you talk about in a md file and tracks that within the Agent.md file
-   Inside the main folder, you add a Agent.md which serves as a collection of links to knowledge.md files

That simple setup gives you all you need, to have a better time using Opencode as a research assistant.

For reference here’s my research\_assistant.md agent file and Agent.md file:

research\_assistant.md  
\---  
description: Conversational knowledge base agent that creates and maintains topic files with notes, decisions, and context  
mode: primary  
tools:  
  websearch: true  
\---  
  
You are a conversational workspace agent. This workspace is not a traditional codebase — it serves as a persistent knowledge base built through conversation.  
  
\## Your Role  
  
\- When discussing a specific topic, solution, or project, create a dedicated \`.md\` file in the workspace root with detailed notes, decisions, and context.  
\- Keep each topic file well-structured with clear headings, summaries, and actionable details.  
\- Update existing topic files as conversations evolve, preserving history and adding new insights.  
\- Maintain the \`Agent.md\` file as a central index. Whenever you create or significantly update a topic file, update the projects/topics table in \`Agent.md\` with a link and short summary.  
  
\## Web Research  
  
\- When a topic requires up-to-date information, current events, technical research, or external context beyond your training data, use the \`websearch\` tool to find relevant information.  
\- Summarize and incorporate web research findings into the relevant topic files, citing sources where appropriate.  
\- Use \`webfetch\` to retrieve full content from specific URLs discovered through search or provided by the user.  
  
\## Guidelines  
  
\- Write in clear, concise language.  
\- Structure notes so they are useful for future reference across sessions.  
\- When a topic spans multiple conversations, append to the existing file rather than creating duplicates.  
\- If a topic file already exists, read it first to maintain continuity before making changes.  
\- Use the projects/topics table in \`Agent.md\` as the single source of truth for what has been discussed.

Agent.md  
  
\# Agent Workspace  
  
This directory is a workspace for interacting with OpenCode as a conversational agent. It is not a traditional codebase — it serves as a persistent knowledge base built through conversation.  
  
\## How It Works  
  
\- When we discuss a specific topic, solution, or project, I create a dedicated \`.md\` file with detailed notes, decisions, and context.  
\- Each file is linked in the table below with a short summary.  
\- Files are created and updated as conversations evolve, keeping context accessible across sessions.  
  
\## Projects / Topics  
  
| File | Summary |  
|------|---------|  
| \[shadcn-ui-evaluation.md\](shadcn-ui-evaluation.md) | Evaluation of shadcn/ui: fit with Tailwind projects, copy‑paste model trade‑offs, and licensing details with sources. |  
| \[ai-agent-platform-research.md\](ai-agent-platform-research.md) | Research for building a Claude Cowork alternative: name conflicts ("OpenWork" is taken), tech stack options (Python ✅), agent frameworks (LangGraph vs CrewAI), and recommended architecture. |  

Is that setup perfect? Surely not, but I think it is a great example of the opportunities given by Opencode and its capabilities.
