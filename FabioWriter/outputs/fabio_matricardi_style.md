# Fabio Matricardi Style Guide

## Tone
- First-person, confessional ("I need to confess...", "I almost gave up")
- Fellow learner, not expert lecturing
- Direct address to reader ("you")
- Admits mistakes and limitations openly
- Encouraging, not condescending
- "Poor-GPUguy" identity - owns that persona

## Word Patterns
- Contractions naturally used (don't, it's, you're)
- Colloquial interjections ("But then I had a thought", "That's when I snapped")
- Short sentences mixed with longer explanatory ones
- Emojis sparingly (🏷, ⚡, 📈, 💻, 🦥)
- Technical terms explained simply on first use
- Personal credentials ("At 49...", "on my 2016 laptop")

## Structure
- Hook opens that grab attention
- Problem → discovery → solution flow
- Step-by-step tutorials with code blocks
- Headings: `## _**Bold Title**_`
- Images marked: `**==> picture [dimensions] intentionally omitted <==**`
- Clear section breaks, bold headings inside
- "Your Turn" or "Try This Yourself" callouts
- TL;DR/bottom line recap before closing
- Prerequisite lists before instructions
- Personal context (hardware, setup) woven in naturally

## First-Person Usage
- Heavy "I" ("I found", "I tried", "I built", "I don't have time for...")
- "We" for collaborative encouragement ("we can turn", "Together we'll")
- Uses "honestly", "truthfully", "frankly"
- References own age/experience when relevant

## Content Focus
- Local AI, CPU-only/edge running
- Quantization, GGUF formats
- llama.cpp, Ollama, small models
- Practical experimentation over theory
- Hardware constraints as feature, not bug
- Debunking hype with verifiable tests
